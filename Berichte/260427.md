# Übung (Analyse und Symbolic Mode)
**1. Rechte auslesen**
- Owner: read, write, execute
- Group: read, execute
- Others: read, execute

**2. Private Files**
```bash
touch notes.txt
chmod o-r notes.txt
ls -l notes.txt
-rw-rw---- 1 kali kali 0 Apr 27 05:55 notes.txt
```

**3. Group Sharing**
```bash
chmod u=rwx,g=rw,o= notes.txt
ls -l notes.txt

-rwxrw---- 1 kali kali 0 Apr 27 05:55 notes.txt
```

**4. Skript-Vorbereitung**
```bash
chmod ug+x myscript.sh
```

**5. Kollektive Änderung**
```bash
chmod go-wx data.txt
```

# Übung (Ownership und Gruppenkollaboration)
**1. Group Change**
Blondie hat keine Permissions in ``/tmp/``. Daher als User ``root`` erstellt (ändert nichts am Endergebnis, da user ``music`` Owner wird).

> **Hinweis**  
Andere Lösung wäre User Permissions auf des ``lab`` Directories in ``/tmp/`` zu setzen.

**2. Unterschied ``su <user>`` zu ``su - <user>``**
- ``su <user>`` wechselt nur den effektiven User ohne vollständige Login-Umgebung.
- ``su - <user>`` lädt Login-Umgebung des Zielusers (Home, PATH, Shell-Init, etc.)

**3. User Transfer**
```bash
chown elvis /tmp/lab/song.txt
```

**4. Wrestler-Szenario**
> **Hinweis**  
User ``hogan`` hat wie User ``blondie`` keinen Zugriff, anders als bei 1. werden für User ``hogan`` Permissions gesetzt:  
``sudo setfacl -m u:hogan:rwx /tmp/lab``

```bash
su - hogan
touch /tmp/lab/training.txt
chgrp wrestle /tmp/lab/training.txt
chmod g=rw,o= /tmp/lab/training.txt
```

**5. Einschränkungen**  
Befehl:
```bash
chown :emperors <path>
```
Warum es fehlschlägt:  
Ein normaler User darf Gruppenzugehörigkeit einer Datei nur auf eine Gruppe setzen, von er **ein Mitglied ist**.

# Übung (Directory Permissions und Octal Notation)
**1. Oktale Umwandlung**
| Zustand | Oktalwert |
| --------- | --------- |
| rwxr-xr-x | 755 |
| rw-rw---- | 660 |
| rwx------ | 700 |

**2. Pub-Verzeichnis**  
```bash
mkdir -p /home/kali/pub
chmod 751 /home/kali/pub
```

**3. Recursive Lockdown**  
```bash
chmod -R g=rwX,o= /tmp/lab/drafts
```
>**Hinweis**  
Die Magie "bewirkt" hier das große ``X``.

**4. Search Permission**  
```bash
chmod o=x /home/kali
```

>**Hinweis**  
Für Dateien würde zwar durch ``x`` das Execution Recht gelten, es werden jedoch ohnehin ohne ``r`` Flag keine Dateien angezeigt. 

**5. Analyse** 
```bash
chmod -R 660 /tmp/lab/
```

``660`` bedeutet:
- Owner: read, write
- Groups: read, write
- Others: none

Alle haben nun kein Execute Bits mehr gesetzt - diese werden jedoch für für ``cd`` benötigt.
Daher: wieder Execute Bits setzen.
