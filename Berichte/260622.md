4. Zeile erkennen und umändern
In meinem Fall ist die zu ändernde Zeile befindet sich bei der Addresse ``0x00005555555551a5 <+69>``. Dort befindet sich der "Jump if equal" Befehl, welcher bei einem Output von ``0`` in eine andere Zeile in der Main Methode springt. Der nächste Befehl beginnt bei ``<+75>``, was bedeutet, dass der ``jei`` Befehl 6 Byte groß ist.
Überschreibt man diese 6 Byte mit ``nop`` - sprich keine Operation, umgeht man die Guard-Clause und das Programm läuft ohne Vergleich weiter.

Setzen der 6 Bytes auf ``nop`` (``0x90``)
```bash
(gdb) set {unsigned char} 0x00005555555551a5 = 0x90
(gdb) set {unsigned char} 0x00005555555551a6 = 0x90
(gdb) set {unsigned char} 0x00005555555551a7 = 0x90
(gdb) set {unsigned char} 0x00005555555551a8 = 0x90
(gdb) set {unsigned char} 0x00005555555551a9 = 0x90
(gdb) set {unsigned char} 0x00005555555551aa = 0x90
```

# Übung (pwn add to sub)
Bei mir findet sich der Ausschnitt mit der Addition unter
```
   0x0000555555555224 <+196>:   add    -0xc(%rbp),%eax
``` 

Aus ``%eax`` ergibt sich, dass es sich um ein 32-Bit Register handelt. Daraufhin muss auch der Hexcode von der 32-Bit Subtraktion verwendet werden. --> Internetsuche: ``0x2b``

>**Wichting**
Hier muss nur das erste Byte abgeändert werden, da meiner Vermutung nach die restlichen 2 Bytes Informationen für die Operation selbst beinhaltet, wie Register etc.

```bash
(gdb) set {unsigned char} 0x0000555555555224 = 0x2b     # ändern auf sub
(gdb) c                                                 # Programm starten
Continuing.
Enter password: G7pL9xQ2mR4tZcH                         # Bereits ermitteltes Passwort verwenden
Correct
Password was: G7pL9xQ2mR4tZcH
a=10
b=5
Result=5                                                # Erfolg!
```
