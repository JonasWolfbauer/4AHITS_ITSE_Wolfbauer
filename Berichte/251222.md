# Vulnerability Scanning
Ziel der Übung ist es ein bestimmtes Zielsystem nach Schwachstellung zu scannen und möglichst viele Infos zu erlangen, welche für einen hypotethischen Angriff von Nöten wären. 

# Übung (``nmap enumeration``)


Es wird ein Command von dem Attack ``kali`` (``172.16.0.3``) auf das Victim ``Metasploitable 2`` (``172.16.0.4``) ausgeführt und analysiert werden.  
Der Command lautet wiefolgt:
```bash
nmap -Pn -sS -sV -sC -oN nmap_default.txt 172.16.0.4
```

## Breakdown
- ``-Pn``  
Überspringt Ping Scan (nimmt an, dass das Ziel online ist)
 - ``-sS``  
``TCP SYN`` Scan. Der Angreifer sendet ``SYN`` Packete, erhält vom Target ``SYN/ACK`` aber sendet keine Antwort zurück. (Gilt als sicherer)
- ``-sV``  
Service Erkennung auf offenen Ports
- ``-sC``  
Führt default Scripts für Vulnerability Scanning aus. Kurz **NSE** für **N**map **S**cripting **E**ngine.
- ``-oN``  
Speichert Output in Textfile ``nmap_default``.

## Art des Scans
``nmap`` führt in diesem Fall einen recht ausgeklügelten Vulnerability Scan aus, welcher in diesem Fall auch schon als intrusive gelten. (Matejka, 22.12.2025)

## Ports
``nmap`` scanned standardmäßig 1000 Ports und da es in diesem Fall nicht weiter speizifiert wurde, trifft dies auch hier zu.

```bash
nmap --script=vuln -p T:21 172.16.0.4
```

Daraufhin startet der Server ein Terminal mit dem Port 6200, mit dem sich ein Angriff dann daraufhin verbinden kann. Die Shell verfügt über ``root`` Level!
