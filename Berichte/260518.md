# Übung (Webserver Directory Permissions II)

Erstellen eines Ordners
```bash
mkdir -p /var/www/data/
```

Gruppe setzen
```bash
sudo chgrp www-data /var/www/data
```

Permissions setzen
```bash
sudo chmod 2774 /var/www/data
```
>**Erklärung**  
Die ``2`` am Anfang ist nötig für das SetGID-Bit.  SetGIB sorgt dafür, dass neu erstelle Dateien / Directories die Gruppe erben
- ``7``  
Owner rwx
- ``7``
Gruppe rwx
``4``  
others r (für ls)

Für folgenden Teil:
```
User der Gruppe www-data können Datei-Inhalte anderer User ansehen aber nicht verändern.
```

```bash
sudo setfacl -d -m u:rwx,g::r,o::0 /var/www/data
```
(Credits: Sonnet 4.6 Adaptive, Anthropic LLC)

**Erklärung**
- ``d``  
Gilt nicht für Directory sondern neu erstellte Dateien
- ``m``  
Modify
- ``u::rwx``  
Owner darf lesen, schreiben und ausführen
- ``g::r``  
Gruppe (www-data) darf lesen
- ``o::0``  
Others dürfen nichts.
