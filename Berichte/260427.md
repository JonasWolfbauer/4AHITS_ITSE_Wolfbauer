# Übung (Analyse und Symbolic Mode)
**1. Rechte auslesen**
- Owner: lesen, schreiben, ausführen
- Group: lesen, ausführen
- Others: lesen, ausführen

**2. Private Files**
```bash
touch notes.txt
chmod o-r notes.txt
ls -l notes.txt
-rw-rw---- 1 kali kali 0 Apr 27 05:55 notes.txt
```

**3. Group Sharing**
```bash
chmod u=rwx,g=rw,o= notes.txt
ls -l notes.txt

-rwxrw---- 1 kali kali 0 Apr 27 05:55 notes.txt
```

**4. Skript-Vorbereitung**
```bash
chmod ug+x myscript.sh
```

**5. Kollektive Änderung**
```bash
chmod go-wx data.txt
```

# Übung (Ownership und Gruppenkollaboration)
**1. Group Change**
Blondie hat keine Permissions in ``/tmp/``. Daher als User ``root`` erstellt (ändert nichts am Endergebnis, da user ``music`` Owner wird).

**2. Unterschied ``su <user>`` zu ``su - <user>``**
- ``su <user>`` wechselt nur den effektiven User ohne vollständige Login-Umgebung.
- ``su - <user>`` lädt Login-Umgebung des Zielusers (Home, PATH, Shell-Init, etc.)
